# Auditoria inicial Maverick — IA-TESTE-3

Data: 2026-06-15
Repositório: `azzacz/IA-TESTE-3`
Status: auditoria inicial, não exaustiva

## Resumo executivo

O repositório está público, é grande e parece ser um fork/base derivada de LiteLLM. Ele pode ser útil como laboratório de gateway de LLMs, mas ainda não deve receber dados reais do Grupo Maverick, credenciais, certificados fiscais, informações financeiras, dados de funcionários, clientes ou pacientes.

Não encontrei evidência imediata de segredo real via busca textual inicial, mas há configurações inseguras para uso direto em produção, especialmente no `docker-compose.yml`, que expõe Postgres em `5432` e usa senha padrão/hardcoded.

## Classificação geral

Risco atual: ALTO para uso em produção.
Risco atual: MÉDIO para laboratório local sem dados reais.

## Achados críticos / altos

### 1. Repositório público

O projeto está público. Isso impede qualquer uso com dados sensíveis ou credenciais reais.

Ação recomendada:

- manter apenas laboratório público;
- criar repositório privado separado para Maverick produção;
- não conectar este repo a ambiente real do posto.

### 2. Credencial padrão no docker-compose

Arquivo: `docker-compose.yml`

Achado:

- `DATABASE_URL` contém senha `dbpassword9090`;
- `POSTGRES_PASSWORD` também usa `dbpassword9090`;
- Postgres exposto em `5432:5432`.

Impacto:

- se usado em servidor exposto, abre superfície de ataque;
- risco de acesso indevido ao banco;
- padrão ruim para produção.

Ação recomendada:

- mover senha para `.env` local;
- criar `.env.example` sem segredo real;
- remover exposição externa de `5432` em produção;
- usar rede interna Docker;
- definir senha forte por ambiente.

### 3. Docker roda como root

Arquivo: `Dockerfile`

Achado:

- builder usa `USER root`;
- runtime também usa `USER root`.

Impacto:

- aumento de dano em caso de exploração do container.

Ação recomendada:

- criar usuário não-root para runtime;
- limitar permissões;
- revisar necessidade real de root.

### 4. Base externa LiteLLM

Arquivo: `README.md` e `pyproject.toml`

Achado:

- README aponta LiteLLM;
- `pyproject.toml` mantém nome `litellm`, autores BerriAI e URLs upstream.

Impacto:

- identidade do projeto não está separada;
- risco de confusão entre fork experimental e produto Maverick;
- antes de customizar, confirmar licença, origem e estratégia.

Ação recomendada:

- manter fork como laboratório;
- criar README Maverick separado;
- não rebatizar pacote principal sem planejamento;
- documentar relação com upstream.
